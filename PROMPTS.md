# 🤖 AI Usage & Prompt Log

This project was developed using a “Human-in-the-Loop” AI-assisted workflow.

AI tools were used to accelerate:
- UI iteration
- brainstorming
- documentation
- component scaffolding
- debugging support

while all major decisions related to:
- architecture
- product scope
- audit logic
- Supabase integration
- UX refinement
- deployment strategy

were manually reviewed and implemented.

The goal was to use AI as a productivity multiplier rather than a replacement for engineering judgment.

---

# 1. Development Philosophy

SpendPilot was intentionally built using a balanced workflow:

## AI-Assisted Tasks
AI tools helped accelerate:
- Tailwind CSS layout generation
- component scaffolding
- copywriting refinement
- debugging suggestions
- markdown documentation
- architecture brainstorming

---

## Human-Controlled Decisions

Critical decisions were manually handled, including:
- project scope control
- frontend architecture
- recommendation logic validation
- Supabase integration
- routing structure
- deployment configuration
- UX consistency
- product positioning

The final implementation prioritized:
- maintainability
- realism
- simplicity
- recruiter-facing polish

over blindly generated complexity.

# 2. Example Prompts Used During Development

## UI & Landing Page Prompt

> “Create a startup-style SaaS landing page for a product called SpendPilot. Use React, Tailwind CSS, and a clean dark-indigo color palette. Include a hero section, feature cards, trust-building copy, and clear call-to-action buttons.”

Purpose:
- accelerate visual prototyping
- establish SaaS-style design direction
- generate initial UI structure

---

## Architecture Planning Prompt

> “I am building a React MVP for an AI spend auditing platform. Suggest a scalable but beginner-friendly folder structure using React Router, Tailwind CSS, and lightweight business logic separation.”

Purpose:
- maintain clean project organization
- avoid overengineering
- improve maintainability

---

## Audit Logic Prompt

> “Write a JavaScript recommendation engine that identifies overlapping AI subscriptions like ChatGPT and Claude, calculates potential savings, and generates deterministic financial recommendations.”

Purpose:
- prototype optimization rules
- brainstorm recommendation logic
- validate realistic savings scenarios

---

## Supabase Integration Prompt

> “Help me implement the smallest possible Supabase integration for storing and retrieving audit reports in a React + Vite application without introducing authentication or backend complexity.”

Purpose:
- satisfy assignment requirements
- preserve MVP simplicity
- maintain deployment stability

---

## Documentation Prompt

> “Generate recruiter-friendly technical documentation for a startup-style SaaS MVP including architecture decisions, product thinking, and engineering tradeoffs.”

Purpose:
- improve project presentation
- communicate engineering reasoning clearly
- document development decisions professionally

# 6. Key Lesson Learned

One major takeaway from the project was:

> AI can dramatically accelerate development speed, but product quality still depends heavily on human prioritization and engineering restraint.

The strongest improvements came not from generating more code, but from:
- refining UX
- simplifying flows
- improving realism
- controlling scope
- polishing the final experience

---

# 7. Final Reflection

SpendPilot was intentionally developed as:
- a realistic startup MVP
- a frontend-first SaaS prototype
- a product-focused engineering assignment

AI tools helped accelerate execution, but the final product direction, tradeoffs, and implementation quality were shaped through manual iteration and decision-making.
